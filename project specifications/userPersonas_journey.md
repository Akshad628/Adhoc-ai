# Document 03 — User Personas & User Journey

**Version:** 1.0

**Status:** Final

---

# 1. Purpose

This document defines every primary user of the ADhoc.ai ecosystem, their goals, responsibilities, pain points, permissions, and expected interactions with the platform.

Understanding these personas ensures that every feature, dashboard, workflow, and AI interaction is designed around real user needs rather than technical implementation alone.

---

# 2. User Hierarchy

The platform consists of seven primary user roles.

```
Super Administrator
        │
        ▼
 Administrator
        │
 ┌──────┼─────────┬────────────┐
 │      │         │            │
Faculty Counsellor Placement Admission
 │
 ├───────────────┐
 │               │

```

Each role has a dedicated dashboard, permissions, workflows, AI capabilities, and analytics.

---

# 3. Persona 01 — Student

## Overview

Students are the primary users of the platform. They interact with ADhoc.ai throughout their educational journey, from admission to placement.

---

## Objectives

Students should be able to:

* Access academic information
* Receive AI guidance
* Communicate with the institution
* Track attendance
* View grades
* Access study materials
* Apply for scholarships
* Prepare for placements
* Manage documents
* Interact using voice or chat

---

## Pain Points

* Difficulty finding accurate information
* Waiting for administrative responses
* Multiple disconnected portals
* Complex placement process
* Lack of career guidance
* Manual document verification

---

## Dashboard Goals

Students should immediately see:

* AI Copilot
* Attendance
* Academic Progress
* Upcoming Events
* Notifications
* Placement Status
* Voice Assistant
* Calendar
* Recent Activities

---

## AI Expectations

The student AI should:

* Answer academic questions
* Explain fee structures
* Recommend courses
* Suggest scholarships
* Assist with placement preparation
* Search institutional knowledge
* Explain policies
* Guide through processes

---

# 16. User Experience Principles

Every user journey should follow these principles:

* Minimal clicks
* Clear navigation
* Fast response times
* Context-aware AI assistance
* Consistent layouts
* Responsive interfaces
* Accessible interactions
* Smooth animations
* Immediate feedback

---

# 17. Navigation Philosophy

Navigation should remain consistent across all dashboards.

Primary Navigation:

* Dashboard
* AI Copilot
* Role-Specific Modules
* Notifications
* Profile
* Settings

Global Components:

* Search
* Command Palette
* Voice Assistant
* User Menu
* Notification Center

---

# 18. Exit Points

Users should be able to:

* Log Out
* Switch Institutions (where applicable)
* Switch Accounts
* End Voice Sessions
* Save Drafts
* Resume Previous Sessions

---

# 19. Acceptance Criteria

A successful user experience is achieved when:

* Every user reaches their primary objective within minimal interactions.
* Navigation is intuitive and role-specific.
* AI assistance is context-aware.
* Dashboards prioritize relevant information.
* Users can complete workflows without external support.
